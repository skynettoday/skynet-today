#### [Anthropic releases Sonnet 4.6](https://techcrunch.com/2026/02/17/anthropic-releases-sonnet-4-6/)
Related:
 * [Claude Sonnet 4.6 model brings ‘much-improved coding skills’ and upgraded free tier](https://9to5mac.com/2026/02/17/claude-sonnet-4-6-model-brings-much-improved-coding-skills-and-upgraded-free-tier/)
 * [Claude Sonnet 4.6 delivers frontier-level AI for free and cheap-seat users](https://www.zdnet.com/article/claude-sonnet-4-6-delivers-frontier-level-ai-for-free-and-cheap-seat-users/)
 * [Anthropic releases Claude Sonnet 4.6, continuing breakneck pace of AI model releases](https://www.cnbc.com/2026/02/17/anthropic-ai-claude-sonnet-4-6-default-free-pro.html)

![](https://techcrunch.com/wp-content/uploads/2025/08/Claude-Chrome-Ext_email-hero-hero.png?w=1200)

Anthropic rolled out Claude Sonnet 4.6, a major upgrade to its midsized model that’s now the default for Free and Pro users across Claude chat, Cowork, and the API, with pricing unchanged. The beta offers a 1 million‑token context window—four times larger than before for Sonnet—capable of holding entire codebases, long contracts, or dozens of papers while sustaining effective long‑context reasoning and long‑horizon planning. Notable gains include improved coding, stronger instruction following, upgraded computer use, better agent planning, and advances in knowledge work and design; in early tests, users preferred Sonnet 4.6 over Sonnet 4.5 about 70% of the time and over Opus 4.5 roughly 60%. Benchmark results set new highs for the tier on OS World (computer use), SWE‑Bench (software engineering), and ARC‑AGI‑2 at 60.4%, though the model still trails leaders like Opus 4.6, Gemini 3 Deep Think, and a refined GPT‑5.2 variant.

Positioned as approaching Opus‑level capability at a more practical price/performance point, Sonnet 4.6 targets most coding and office workflows and is often faster than Opus 4.6. The release expands free‑tier features such as file creation, connectors, skills, and compaction, while Anthropic reports fewer instances of overengineering, hallucinations, false success claims, and “laziness,” yielding more reliable multi‑step follow‑through. Computer use remains below top human performance, but Anthropic cites rapid progress and greater utility for real tasks; Opus 4.6 is still recommended for the deepest reasoning, large‑scale refactoring, multi‑agent coordination, and high‑precision work. Arriving just 12 days after Opus 4.6, the Sonnet update continues Anthropic’s roughly four‑month refresh cadence and suggests a Haiku update may be next.

#### [Google Rolls Out Latest AI Model, Gemini 3.1 Pro](https://www.cnet.com/tech/services-and-software/google-announces-gemini-3-1-pro/)
Related:
 * [Google releases Gemini 3.1 Pro: Benchmarks, how to try it](https://mashable.com/article/google-releases-gemini-3-1-pro-benchmarks)

![](https://www.cnet.com/a/img/resize/5c2e8f531bda452077ad64d1b87bd0bc4dd5371f/hub/2026/01/26/de109626-806c-4217-b81a-78bd18728af1/gettyimages-2255802528.jpg?auto=webp&fit=crop&height=675&width=1200)

Google introduced Gemini 3.1 Pro as its new “core reasoning” model, citing large jumps on hard logic and knowledge tests alongside showy creative demos. On ARC‑AGI‑2, it scores 77.1% (up from 31.1% on Gemini 3 Pro), with 94.3% on GPQA Diamond and 92.6% on MMLU. The company also reports 44.4% on “Humanity’s Last Exam,” edging Claude Opus 4.6 (40.0%) and GPT‑5.2 (34.5%). Beyond metrics, demonstrations include code‑based animations (lossless‑scaling SVGs), rapid creative coding (for example, a full Wuthering Heights‑themed website), and interactive design (a controllable 3D starling murmuration with reactive sound).

Availability is expanding across Google’s stack: the Gemini app (free tier with higher limits for AI Pro/Ultra), NotebookLM (AI Pro/Ultra only for 3.1 Pro), and developer/enterprise access via Gemini API in AI Studio, Vertex AI, Gemini Enterprise, Gemini CLI, Antigravity, and Android Studio. Google frames 3.1 Pro as the backbone for Gemini features like Gemini 3 Deep Think—used where “a simple answer isn’t enough.” In head‑to‑head results Google shared, Gemini 3.1 Pro outperforms Claude Sonnet/Opus 4.6 and GPT‑5.2 on most reasoning benchmarks, while OpenAI’s GPT‑5.3‑Codex leads on SWE‑Bench Pro, and GPT‑5.2 slightly tops 3.1 Pro on that public SWE‑Bench Pro metric (55.6% vs. 54.2%). Coding remains strong overall (80.6% on SWE‑Bench Verified vs. 76.2% for Gemini 3 Pro), with Google emphasizing better practicality for complex synthesis and visualization.

#### [OpenAI picks up pace against Claude Code with new Codex desktop app](https://arstechnica.com/ai/2026/02/openai-picks-up-pace-against-claude-code-with-new-codex-desktop-app/)
![](https://cdn.arstechnica.net/wp-content/uploads/2026/02/Codex-dark-1152x648-1770052639.jpg)

OpenAI released a native macOS desktop app for Codex, its LLM‑based coding tool previously limited to a web CLI and IDE extensions. Built to manage multiple long‑running coding agents simultaneously, the app lets tasks run in parallel for hours—something OpenAI argues is clunky in a terminal or IDE sidebar. Agents are organized by project so users can work across several efforts at once; Codex also supports worktrees to isolate changes and prevent conflicts. New features include first‑class support for Skills—folder‑based extensions with instructions and resources—and Automations, which execute scheduled tasks using those Skills.

This move better positions Codex against Anthropic’s Claude Code macOS app, an area where OpenAI had been lagging on features and packaging. To close the gap, OpenAI is doubling Codex rate limits for Plus, Pro, Business, Enterprise, and Edu plans, and temporarily extending Codex access to ChatGPT Free and Go subscribers (with unspecified limits). Early impressions indicate the desktop app meaningfully improves multi‑agent orchestration and long‑running workflows compared with the CLI and IDE integrations.

#### Research

[SkillsBench: Benchmarking How Well Agent Skills Work Across Diverse Tasks](https://arxiv.org/abs/2602.12670). This benchmark measures how curated versus self‑generated Skills affect agent success across 84 terminal‑based tasks, identifying which Skill components and harness behaviors drive gains, common failure modes, and practical authoring guidelines.

[SpargeAttention2: Trainable Sparse Attention via Hybrid Top-k+Top-p Masking and Distillation Fine-Tuning](https://arxiv.org/abs/2602.13515). The authors introduce a hybrid Top‑k+Top‑p masker and a velocity‑level distillation fine‑tuning procedure that enable trainable sparse attention to reach up to 95% sparsity with large runtime speedups, while preserving full‑attention generation quality even when fine‑tuning data differs from pre‑training data.

[Think Deep, Not Just Long: Measuring LLM Reasoning Effort via Deep-Thinking Tokens](https://arxiv.org/abs/2602.13517). This paper defines and validates the deep‑thinking ratio (DTR)—the share of tokens whose predicted distributions only stabilize in later layers—as a layer‑wise, computation‑rooted metric that correlates with reasoning accuracy and enables cheaper, DTR‑guided parallel inference such as Think@nn.

[BitDance: Scaling Autoregressive Generative Models with Binary Tokens](https://arxiv.org/abs/2602.14041). The model represents images with compact binary visual tokens and combines autoregressive and diffusion techniques to generate high‑resolution images more efficiently.

[WebWorld: A Large-Scale World Model for Web Agent Training](https://arxiv.org/abs/2602.14721). Trained on over 1M real‑world web interaction trajectories using a hierarchical data pipeline, WebWorld enables long‑horizon, multi‑format simulation and reasoning, improving agent performance and generalizing across web, code, GUI, and game tasks.

[On Surprising Effectiveness of Masking Updates in Adaptive Optimizers](https://arxiv.org/abs/2602.15322v1). The authors show that randomly skipping and selectively scaling block‑wise gradient updates (via a method called Magma) can improve training stability and generalization by implicitly regularizing against sharp, high‑variance directions, enabling larger stable step sizes and better empirical performance than dense adaptive optimizers.

[Team of Thoughts: Efficient Test-time Scaling of Agentic Systems through Orchestrated Tool Calling](https://arxiv.org/abs/2602.16485). This system treats diverse LLMs as specialized, on‑call tools coordinated by an orchestrator that selectively invokes and budgets agents via native tool‑calling—boosting reasoning and code generation while reducing redundant token output.

[NESSiE: The Necessary Safety Benchmark -- Identifying Errors that should not Exist](https://arxiv.org/abs/2602.16756). NESSiE offers a compact suite of abstract, keyword‑matchable tests—covering reformulations, multi‑turn and agentic behaviors, and paired cases requiring both helpful responses and deliberate withholding—to quickly flag instruction‑following errors unacceptable in safety‑critical deployments.

[2Mamba2Furious: Linear in Complexity, Competitive in Accuracy](https://arxiv.org/abs/2602.17363). Building on Mamba‑2, the approach introduces higher‑order hidden states and an exponentiated query‑key inner product to push linear‑attention accuracy toward (and sometimes beyond) softmax levels while preserving linear training complexity.
